Using URL Parameters

You can open this page with predefined controls by appending query-string parameters to the URL. Use ?param=value for the first option and &param=value for each additional option. For shared links, prefer the canonical parameter names listed below.

Common examples

  • ?xaxis_type=color&yaxis_type=absolute_magnitude&xaxis_value_1=mko_jmag&xaxis_value_2=mko_kmag&yaxis_value_1=mko_jmag&moca_oid=602,10995
  • ?xaxis_type=spectral_type&yaxis_type=absolute_magnitude&yaxis_value_1=mko_jmag&moca_oid=602
  • ?xaxis_type=spectral_type&yaxis_type=spectral_index&yaxis_value_1=h2o_j&spt_range=L2+
  • ?mock=1 for a network-free local sample dataset.

Axis parameters

ParameterAllowed valuesEffect
xaxis_type, yaxis_type spectral_type, color, absolute_magnitude, spectral_index, equivalent_width Chooses the observable shown on each axis. Defaults are xaxis_type=color and yaxis_type=absolute_magnitude.
xaxis_value_1, yaxis_value_1 Photometry system ID, spectral-index ID, or chemical-species ID Chooses the main quantity for axes that need one. Simplified photometry values use simple:J, simple:H, simple:K, or simple:W2. Advanced photometry values use detailed IDs such as mko_jmag, mko_hmag, mko_kmag, wise_w1, and wise_w2. For spectral indices, use IDs such as hcont or h2o_j. For equivalent widths, use a chemical-species ID such as li.
xaxis_value_2, yaxis_value_2 Photometry system ID Chooses the second band for color axes. For example, xaxis_value_1=mko_jmag&xaxis_value_2=mko_kmag plots J-K.

Catalog and filter parameters

ParameterValuesEffect
spt_range Spectral type range such as L2+, >=L2, or M6-Y2 Limits the plotted catalog by adopted spectral type. An underscore may be used instead of a hyphen in ranges. Highlighted OIDs are still requested even when they fall outside the range. Default is L2+.
moca_oid Comma-separated OIDs, for example 602,10995 Highlights specific objects and requests them from the backend even when they are outside the current spectral range or hidden optional filters.
designation One designation per parameter, or comma-separated designations Preselects designation-based highlights for objects already loaded in the current catalog. Use repeated parameters for several names, for example ?designation=WISEA%20J0000&designation=2MASS%20J0001.
oid Comma-separated OIDs Alias for moca_oid.
xerr_max, yerr_max Number, or blank/none/null/nan De-emphasizes points whose propagated X or Y uncertainty exceeds the threshold and excludes those points from automatic axis limits. If omitted, the app applies axis-specific defaults where available.
max_objects Positive integer, or 0/none/uncapped/all Controls the safety cap for broad source queries, especially when including photometric spectral types or requesting spectral ranges earlier than L0. The default broad-query cap is 1,000,000 objects; 0, none, uncapped, or all explicitly requests no cap.

Boolean display options

Boolean options are enabled with 1, true, or yes. Omit the parameter, or use 0 or false, to leave the option off.

ParameterDefaultEffect
errors false Shows propagated X and Y error bars.
photdist false Includes objects with photometric distance estimates when an absolute-magnitude axis is active.
binaries false Plots known binaries. They are drawn with open marker symbols and listed in the legend.
photspt false Includes photometric spectral type estimates. This can be slow because it broadens the source query.
advanced_photometry false Shows all detailed MOCAdb photometry system IDs instead of the simplified band list.
agecolor false Colors objects with BANYAN association ages by age and shows objects without ages in gray.

Backend aliases

The backend also recognizes include_photdist, include_photometric_dist, include_photspt, and include_photometric_spt. The visible page controls are synchronized to the canonical photdist and photspt parameters, so those canonical names are preferred for URLs you intend to share.

Performance Preload

For the current spectral range and highlighted objects, this preloads the full in-memory dataset: photometric spectral type estimates, photometric distances, designations, all photometry bands, all spectral indices, equivalent widths, age data, and sequence overlays. This can take a few minutes, but once it finishes the app can switch most axes and display options much faster. Some large optional private datasets may be omitted from this upfront load to keep the preload practical.
